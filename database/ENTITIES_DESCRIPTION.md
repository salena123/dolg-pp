# Описание сущностей базы данных

В проекте используется **10 сущностей** (таблиц) в базе данных.

## Используемые сущности (7 основных)

### 1. **departments** (Отделы)
**Назначение:** Хранит информацию об отделах университета (факультетах, кафедрах, административных подразделениях).

**Поля:**
- `id` - уникальный идентификатор
- `name` - название отдела
- `office` - номер кабинета/офиса
- `phone` - контактный телефон

**Использование:** Работодатели привязываются к отделам. Позволяет организовать структуру университета.

---

### 2. **users** (Пользователи)
**Назначение:** Хранит учетные данные всех пользователей системы (студентов, работодателей, администраторов).

**Поля:**
- `id` - уникальный идентификатор
- `name` - имя пользователя
- `email` - электронная почта (уникальная)
- `password_hash` - хеш пароля
- `role` - роль пользователя (student, employer, admin)
- `created_at` - дата регистрации

**Использование:** Основа системы аутентификации и авторизации. Все действия в системе привязаны к пользователям.

---

### 3. **employers** (Работодатели)
**Назначение:** Хранит профили работодателей (преподавателей, администраторов, руководителей проектов).

**Поля:**
- `id` - уникальный идентификатор
- `user_id` - связь с пользователем (один к одному)
- `name` - название/имя работодателя
- `department_id` - отдел, к которому относится работодатель
- `contact_email` - контактный email
- `description` - описание работодателя

**Использование:** Работодатели создают вакансии и управляют заявками студентов. Связаны с пользователями и отделами.

---

### 4. **jobs** (Вакансии)
**Назначение:** Хранит информацию о доступных вакансиях и стажировках.

**Поля:**
- `id` - уникальный идентификатор
- `employer_id` - работодатель, создавший вакансию
- `title` - название вакансии
- `description` - описание вакансии
- `location` - местоположение
- `employment_type` - тип занятости (full-time, part-time, internship, contract)
- `remote` - удаленная работа (да/нет)
- `start_date` - дата начала
- `end_date` - дата окончания
- `spots` - количество мест
- `status` - статус (open, closed)

**Использование:** Основная сущность для поиска работы. Студенты просматривают вакансии и подают на них заявки.

---

### 5. **applications** (Заявки)
**Назначение:** Хранит заявки студентов на вакансии.

**Поля:**
- `id` - уникальный идентификатор
- `job_id` - вакансия, на которую подана заявка
- `user_id` - студент, подавший заявку
- `status` - статус заявки (submitted, reviewed, accepted, rejected)
- `cover_letter` - сопроводительное письмо
- `resume_url` - ссылка на резюме
- `submitted_at` - дата подачи заявки
- `updated_at` - дата последнего обновления

**Использование:** Студенты подают заявки, работодатели их просматривают и меняют статус. Основа процесса найма.

---

### 6. **reviews** (Отзывы студентов)
**Назначение:** Хранит отзывы студентов о вакансиях и стажировках.

**Поля:**
- `id` - уникальный идентификатор
- `job_id` - вакансия, на которую оставлен отзыв
- `employer_id` - работодатель
- `user_id` - студент, оставивший отзыв
- `rating` - оценка (1-5)
- `comment` - текстовый комментарий
- `created_at` - дата создания отзыва

**Использование:** Студенты оставляют отзывы о пройденных стажировках. Помогает другим студентам выбирать вакансии.

---

### 7. **employer_reviews** (Отзывы работодателей)
**Назначение:** Хранит отзывы работодателей о студентах после принятия заявки.

**Поля:**
- `id` - уникальный идентификатор
- `application_id` - заявка, по которой оставлен отзыв
- `job_id` - вакансия
- `student_id` - студент, о котором отзыв
- `employer_id` - работодатель, оставивший отзыв
- `rating` - оценка (1-5)
- `comment` - текстовый комментарий
- `created_at` - дата создания отзыва

**Использование:** Работодатели оценивают работу студентов после принятия заявки. Помогает формировать репутацию студентов.

---

## Резервные сущности (3 для будущего использования)

### 8. **favorites** (Избранное)
**Назначение:** Позволяет студентам сохранять понравившиеся вакансии в избранное.

**Поля:**
- `id` - уникальный идентификатор
- `user_id` - студент
- `job_id` - вакансия
- `created_at` - дата добавления в избранное

**Использование:** Пока не реализовано в коде, но логически связано с основной функциональностью. Может быть добавлено в будущем для удобства студентов.

---

### 9. **interviews** (Собеседования)
**Назначение:** Хранит информацию о запланированных собеседованиях.

**Поля:**
- `id` - уникальный идентификатор
- `application_id` - заявка, по которой назначено собеседование
- `scheduled_at` - дата и время собеседования
- `location` - место проведения
- `mode` - формат (очно, онлайн)
- `notes` - заметки

**Использование:** Пока не реализовано в коде, но логически связано с процессом найма. Может быть добавлено для управления календарем собеседований.

---

### 10. **attachments** (Вложения)
**Назначение:** Хранит дополнительные файлы, прикрепленные к вакансиям (документы, инструкции, примеры работ).

**Поля:**
- `id` - уникальный идентификатор
- `job_id` - вакансия, к которой прикреплен файл
- `file_url` - ссылка на файл
- `description` - описание файла

**Использование:** Пока не реализовано в коде, но может быть полезно для работодателей, которые хотят прикрепить дополнительные материалы к вакансии.

---

## Итого

- **Всего сущностей:** 10
- **Активно используемых:** 7 (departments, users, employers, jobs, applications, reviews, employer_reviews)
- **Резервных для будущего:** 3 (favorites, interviews, attachments)

Все сущности логически связаны с основной функциональностью системы поиска работы и стажировок в кампусе.
